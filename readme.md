## 🛠️ Project Overview - Learning C++ & Building a Game Engine from Scratch

This repository documents my journey of learning **modern C++** and **low-level graphics programming** while working toward a long-time personal dream:  
**building a fully functional game engine, and later, a small game on top of it.**

C++ is not an easy language for me.  
I come from higher-level environments, and learning C++ means learning how systems really work: memory ownership, lifetimes, resource management, and explicit control over hardware. It is often frustrating, slow, and mentally demanding - but also deeply rewarding.

---

## 🎯 Why this project exists

I am not pursuing a professional career in game development.

This project exists because:
- building a game engine was my dream since I touched a computer for the first time
- understanding how engines, GPUs, and real-time systems work is **golden knowledge**
- I want to challenge myself far beyond tutorials and abstractions
- I believe struggling with hard systems builds lasting understanding

This repository is **not a polished engine**, and it is not meant to be.  
It is a **learning log**, a **technical playground**, and a **record of progress**.

---

## 🧠 What I am learning here

- Modern **C++** (RAII, ownership, lifetimes, containers)
- **Vulkan** (explicit GPU control, queues, swapchains, synchronization)
- **SDL2** (windowing, input, platform abstraction)
- Real-time programming concepts (delta time, frame loops, resource management)
- Engine architecture (separation of engine vs game logic)

Mistakes, refactors, and unfinished systems are part of the process.

---

## 📌 Project philosophy

- No magic
- No hidden abstractions
- Learn by building, breaking, and fixing
- Prefer understanding over speed
- Document progress honestly

---

## 🚧 Current state

The project is under active development and learning.  
Expect:
